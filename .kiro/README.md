# Kiro Configuration for TNR Tracker

This folder contains Kiro-specific configuration files that help guide AI-assisted development without modifying the actual application code.

## Folder Structure

- **`steering/`** - Contains context and guidelines that Kiro uses when working on this project
  - `project-overview.md` - High-level project information and structure
  - `development-guidelines.md` - Best practices and safety guidelines
  
- **`settings/`** - Contains Kiro configuration files (e.g., MCP server settings)

## Purpose

These files help Kiro:
- Understand your project structure and architecture
- Follow your team's coding standards and conventions
- Avoid making changes that could break working functionality
- Work more efficiently by having context about the project

## Note

The `.kiro` folder is for tooling configuration only and does not affect your application's runtime behavior.
